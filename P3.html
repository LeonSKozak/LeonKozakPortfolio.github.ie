Python Project 3
Sentiment Analysis

In this project, Lasse Willen, Vincent Stilz and I worked on a data set containing 568,264 rows of sentiment Data. In this case, the sentiment is based on online reviews about different products. The data set contains collumns such as Profile Name, A ratig of the product (1-5 stars, 5 being best), and the review itself in text form.

Below, you can take a look at the data set. Since the original data set contains over 560,000 rows, the version below is shortened to 1,500 columns, as Google Tables would not be able to showcase it otherwise.

First, we quickly examined the data. This showed us, that the data is heavily leaning towards positive product ratings. Next, we looked at one example review and tokenized it, using NLTK. In the next step, we used Vader Sentiment Scoring. First, we tested the vader scoring on some very explicit example sentences, such as "I am so happy". This testing helped us to get a feeling for how this scoring works. Then, we let the polarity score, which gave each review a score for negativity, neutrality, positivity and then a compounded score, run over the entire data set. This showed us, that the higher the polarity score, the higher the product rating (Score). Furthermore, we used a pre-trained tokenizer (Roberta Model), which was trained on large quantities of twitter data. We then compared the scores of the Vader Sentiment Scores with the Roberta Sentiment Scores. Ultimately, we checked for incorrect sentiment scores, meaning cases, in which a high score was given by the user, but our models thought that the review was negative and vice versa. This showed us, that in certain cases, sentiment analysis is still flawed. Still, for large data sets, it is able to detect sentiment correctly.
